1 · System Overview — comma.ai Controls Challenge

The comma.ai Controls Challenge asks participants to design a lateral acceleration (lataccel) tracking controller for an autonomous driving scenario. A black-box neural physics model called TinyPhysics plays the role of the vehicle dynamics plant. The controller must issue a steer command at each timestep so that the vehicle's simulated lateral acceleration tracks a reference trajectory.

Three controllers have been implemented and benchmarked in this codebase:

ControllerMethodLatAccel CostJerk CostTotal Cost

Score Formula: Total Cost = (LatAccel Cost × 50) + Jerk Cost. Lower is better. The ×50 weight makes tracking accuracy far more important than smoothness.

2 · TinyPhysics Simulator

2.1 State & Action Space

Each CSV data file contains one driving segment. The simulator reads the following signals:

SignalDescriptionUnits
roll_lataccelLateral acceleration due to road roll = sin(roll) × gm/s²
v_egoVehicle longitudinal velocitym/s
a_egoVehicle longitudinal accelerationm/s²
target_lataccelReference lateral acceleration to trackm/s²
steer_commandSteering input (right-positive convention)normalised [−2, 2]

The state observed by the controller at each step is wrapped in a State namedtuple, and a 5-second preview (50 steps at 10 Hz) is available as a FuturePlan.

2.2 Neural Dynamics Model

TinyPhysics uses a pre-trained ONNX transformer/token model as the vehicle physics engine. At each step it predicts the next lateral acceleration token from the last 20 context steps of states, actions, and past predictions, then decodes the token through a 1024-bin tokenizer that linearly covers [−5, 5] m/s².

Tokenizer decoding
\[\hat{y}_t = \text{bins}\!\left[\arg\max_k \frac{e^{l_k / T}}{\sum_j e^{l_j/T}}\right], \quad T = 0.8, \quad \text{bins}_k = -5 + \frac{10k}{1023}\]

The predicted acceleration is then clipped to prevent unrealistic jumps:

Continuity constraint
\[\hat{y}_t = \text{clip}\!\left(\hat{y}_t,\; y_{t-1} - 0.5,\; y_{t-1} + 0.5\right)\]

2.3 Cost Functions

Costs are computed over control steps 100 to 500 (the first 100 steps use recorded actions as warm-up).

Lateral acceleration tracking cost
\[C_\text{lat} = \frac{100}{N} \sum_{t=100}^{500} \left(r_t - \hat{y}_t\right)^2\]
Jerk cost
\[C_\text{jerk} = \frac{100}{N} \sum_{t=101}^{500} \left(\frac{\hat{y}_t - \hat{y}_{t-1}}{\Delta t}\right)^2, \quad \Delta t = 0.1\text{ s}\]
Total cost (objective to minimise)
\[C_\text{total} = 50 \cdot C_\text{lat} + C_\text{jerk}\]

2.4 Simulation Loop

Initialise
Load CSV, seed RNG from file hash, replay first 20 steps (context window) with recorded actions.
↓
Each Step t
Read state (vEgo, aEgo, roll) and target from CSV. Assemble FuturePlan (next 50 steps).
↓
Control Step
Call controller.update(target, current_lataccel, state, future_plan) → steer ∈ [−2, 2].
↓
Simulation Step
Push steer to TinyPhysics ONNX model → sample next lataccel token → decode → clip.
↓
Cost Computation (after all steps)
Compute C_lat, C_jerk, C_total over steps 100–500.

3 · Koopman Operator Theory

3.1 The Koopman Operator

For a discrete nonlinear dynamical system \(x_{t+1} = f(x_t)\), the Koopman operator \(\mathcal{K}\) is an infinite-dimensional linear operator that acts on observable functions \(g : \mathcal{X} \to \mathbb{R}\):

Koopman operator definition
\[\mathcal{K}\, g = g \circ f \qquad \Longleftrightarrow \qquad \mathcal{K}\, g(x_t) = g(f(x_t)) = g(x_{t+1})\]

The key insight: while \(f\) is nonlinear, \(\mathcal{K}\) is linear — nonlinear dynamics become linear evolution in the (possibly infinite-dimensional) space of observables.

3.2 Finite-Dimensional Approximation

We approximate \(\mathcal{K}\) by choosing a finite dictionary of basis functions (observables) \(\phi : \mathbb{R}^n \to \mathbb{R}^{n_z}\) and finding the best linear map in that lifted space:

Lifted state
\[z_t = \phi(x_t) \in \mathbb{R}^{n_z}, \qquad n_z \gg n\]
Lifted linear dynamics (Koopman model)
\[z_{t+1} \approx A\,z_t + B\,u_t, \qquad A \in \mathbb{R}^{n_z \times n_z},\quad B \in \mathbb{R}^{n_z \times n_u}\]
Output readout
\[\hat{y}_t = C\,z_t, \qquad C \in \mathbb{R}^{n_y \times n_z}\]

3.3 Why Koopman for Control

Once in the lifted linear space, the full machinery of linear MPC applies: closed-form predictions, guaranteed stability via spectral radius control, and efficient optimisation. The quality of the approximation depends on the richness of the dictionary \(\phi\).

4 · Koopman Training Pipeline (koopman_trainingv3.py)

Summary: The training script learns A, B, C matrices that define a 90-dimensional linear system approximating the nonlinear vehicle lataccel dynamics. Training uses 900 CSV segments (≈518,793 state-transition pairs).

4.1 Data Loading & Preprocessing

Each CSV file contains a driving segment with the raw state columns:

State vector (raw, pre-scaling)
\[x_t = \begin{bmatrix} v_\text{ego} \\ a_\text{ego} \\ \text{roll} \\ \text{current\_lataccel} \end{bmatrix} \in \mathbb{R}^4\]

900 files are loaded for training and 100 for validation. The control input is:

Control input
\[u_t = \text{steerCommand} \in \mathbb{R}\]

4.2 Feature Scaling

A StandardScaler is fit on the full training set and applied to both states and controls. This is critical because the raw features have very different magnitudes (velocity in m/s vs. lataccel in m/s²).

Z-score normalisation
\[\tilde{x}_t = \frac{x_t - \boldsymbol{\mu}_x}{\boldsymbol{\sigma}_x}, \qquad \tilde{u}_t = \frac{u_t - \mu_u}{\sigma_u}\]

where \(\boldsymbol{\mu}_x, \boldsymbol{\sigma}_x \in \mathbb{R}^4\) and \(\mu_u, \sigma_u \in \mathbb{R}\) are fit on training data only.

4.3 Delay Embedding

A single 4-dimensional snapshot gives limited temporal information. Delay embedding stacks the last \(d = 3\) consecutive scaled state vectors into a single expanded vector, injecting temporal context (Takens' theorem motivates this for reconstructing attractor geometry).

Delay-embedded state (DELAY = 3)
\[\tilde{x}^{(d)}_t = \begin{bmatrix} \tilde{x}_t \\ \tilde{x}_{t-1} \\ \tilde{x}_{t-2} \end{bmatrix} \in \mathbb{R}^{4d} = \mathbb{R}^{12}\]

In code this is computed in reverse order then reshaped, so the most recent state occupies the first 4 entries. This requires \(d-1 = 2\) warm-up steps, reducing each trajectory by 2 samples from the start.

4.4 Polynomial Lifting

The delay-embedded vector is further lifted using a degree-2 polynomial feature map. This is the dictionary \(\phi\) that generates all monomials up to degree 2:

Polynomial lifting map (degree 2, no bias)
\[z_t = \phi\!\left(\tilde{x}^{(d)}_t\right) = \left[\tilde{x}^{(d)}_t,\; \left(\tilde{x}^{(d)}_t\right)^{\otimes 2}_{\text{upper}}\right] \in \mathbb{R}^{n_z}\]

For an input of dimension \(p = 12\) with degree 2 and no bias, the output dimension is:

Lifted dimension formula
\[n_z = p + \binom{p+1}{2} = 12 + \frac{12 \cdot 13}{2} = 12 + 78 = \mathbf{90}\]

This is confirmed by the training output: "Lifted dim: 90 (delay=3, poly_deg=2)".

4.5 Building Training Pairs

For each trajectory we build one-step pairs \((z_t, u_t, z_{t+1})\) along with the output targets:

Training pair construction
\[\{(z_k, u_k, z_{k+1},\; y_k,\; \Delta y_k)\}_{k}\] \[\text{where}\quad y_k = \text{lataccel at step } k, \qquad \Delta y_k = y_{k+1} - y_k\]

The total number of pairs across 900 trajectories is 518,793.

4.6 Fitting Koopman Matrices A and B (Ridge Regression)

The Koopman dynamics \(z_{t+1} = Az_t + Bu_t\) are fit jointly by stacking \([z_k,\, u_k]\) as a design matrix and solving with Ridge regularisation:

Joint design matrix
\[X = \begin{bmatrix} Z_k & U_k \end{bmatrix} \in \mathbb{R}^{N \times (n_z + 1)}\]
Ridge regression (normal equations)
\[K = \left(X^\top X + \lambda I\right)^{-1} X^\top Z_{k+1}, \qquad \lambda = 0.05\]
Extract A and B from solution
\[A = K_{1:n_z}^\top \in \mathbb{R}^{n_z \times n_z}, \qquad B = K_{n_z+1}^\top \in \mathbb{R}^{n_z \times 1}\]

Ridge regularisation (\(\lambda = 0.05\)) prevents overfitting and improves the condition number of \(X^\top X\), which is critical for a 90-dimensional system with correlated features.

4.7 Dual-Head Output Readout (Fitting C)

Instead of a single output matrix, two independent readout heads are fitted:

Absolute prediction head
\[W_\text{abs} = \left(Z_k^\top Z_k + \lambda I\right)^{-1} Z_k^\top Y_\text{abs} \in \mathbb{R}^{n_z \times 1}\]
Delta prediction head
\[W_\Delta = \left(Z_k^\top Z_k + \lambda I\right)^{-1} Z_k^\top Y_\Delta \in \mathbb{R}^{n_z \times 1}\]
Combined C matrix
\[C = \begin{bmatrix} W_\text{abs}^\top \\ W_\Delta^\top \end{bmatrix} \in \mathbb{R}^{2 \times n_z}, \quad C[0] = W_\text{abs}^\top,\; C[1] = W_\Delta^\top\]

Fitting the two heads separately avoids cross-interference: the absolute head learns the steady-state lataccel from any lifted state; the delta head learns incremental changes. Both are then blended at inference time.

4.8 Spectral Stabilisation

The learned A matrix may have eigenvalues with magnitude exceeding 1, causing unbounded rollouts. To guarantee stability, eigenvalues are scaled down to a maximum spectral radius of \(\rho_\max = 0.97\):

Eigendecomposition
\[A = V \Lambda V^{-1}, \qquad \Lambda = \text{diag}(\lambda_1, \ldots, \lambda_{n_z})\]
Eigenvalue scaling
\[\tilde{\lambda}_i = \begin{cases} \dfrac{\rho_\max}{|\lambda_i|} \cdot \lambda_i & \text{if } |\lambda_i| > \rho_\max \\ \lambda_i & \text{otherwise} \end{cases}\]
Stabilised A matrix
\[A_\text{stab} = V\,\tilde{\Lambda}\,V^{-1} \in \mathbb{R}^{n_z \times n_z}, \qquad \tilde{\Lambda} = \text{diag}(\tilde{\lambda}_1, \ldots, \tilde{\lambda}_{n_z})\]

Result from training: "Clipped 9/90 eigenvalues (max |λ| before=0.9997, after=0.9700)". Only 9 of 90 modes needed correction — the data-driven A is already mostly stable.

4.9 Blended Horizon Evaluation

After fitting, a H=15 open-loop rollout evaluates model quality. Each rollout step uses the blended prediction formula:

Koopman state rollout
\[z_{t+1} = A_\text{stab}\,z_t + B\,\tilde{u}_t\]
Blended output prediction (α = 0.80)
\[\hat{y}_{t+1} = \alpha \!\left(\hat{y}_t + C[1]\cdot z_t\right) + (1-\alpha)\!\left(C[0]\cdot z_t\right)\]

This blending formula combines two philosophies:

  • Absolute (1−α = 20%): predicts lataccel directly from the lifted state — good for recovering after large deviations.
  • Delta (α = 80%): integrates incremental changes from the current lataccel — good for smooth short-horizon prediction.
Evaluation metric
\[R^2 = 1 - \frac{\sum_t (\hat{y}_t - y_t)^2}{\sum_t (y_t - \bar{y})^2}\]

Results: Train R² = 0.8703, Val R² = 0.8815 (validation slightly outperforming training is normal with ridge regularisation).

5.1 Model Loading & State Buffer

At import time, the trained artefacts are loaded from disk. The controller maintains a circular state buffer of length DELAY = 3. Until 3 steps have been observed, a simple feedforward fallback is used:

Warm-up feedforward (before state buffer filled)
\[u_t^{\text{warmup}} = \text{FF\_GAIN} \cdot r_t, \qquad \text{FF\_GAIN} = 0.31498\]

5.2 Lifting the Current State

Each call to update() appends the new state to the buffer, then lifts the 3-step window to obtain the current Koopman state vector:

State lifting pipeline
\[x_\text{hist} \xrightarrow{\text{StandardScaler}} \tilde{x}_\text{hist} \xrightarrow{\text{reverse+reshape}} \tilde{x}^{(d)} \xrightarrow{\phi(\cdot)} z_t \in \mathbb{R}^{90}\]

5.3 Random-Shooting MPC Optimisation

MPC is solved by evaluating a finite set of constant-hold steer candidates over an H=8 horizon and selecting the one with minimum cost. The candidate set is:

Candidate generation
\[\mathcal{U} = \text{linspace}(u_{t-1} - 0.2,\; u_{t-1} + 0.2,\; 15) \;\cup\; \text{linspace}(u_{t-1} - \sigma,\; u_{t-1} + \sigma,\; 15)\] \[\sigma = 0.6 + 0.8\,|u_{t-1}|, \qquad \mathcal{U} \subset [-2, 2]\]

For each candidate \(u \in \mathcal{U}\), the MPC rollout proceeds:

Koopman rollout (inner loop, k = 0, …, H−1)
\[\tilde{u} = \text{u\_scaler.transform}(u) \qquad \text{(scale to training distribution)}\] \[z_{k+1} = A_\text{stab}\,z_k + B\,\tilde{u}\] \[\hat{y}_{k+1} = \alpha\!\left(\hat{y}_k + C[1] \cdot z_{k+1}\right) + (1-\alpha)\!\left(C[0]\cdot z_{k+1}\right)\]
Per-step cost
\[c_k(u) = \underbrace{\left(\hat{y}_{k+1} - r_{k+1}\right)^2}_{\text{tracking}} + \underbrace{\lambda_\text{jerk}\,(u - u_\text{last})^2}_{\text{smoothness}} + \underbrace{\lambda_u\, u^2}_{\text{effort}}\]
Total candidate cost and optimal selection
\[J(u) = \sum_{k=0}^{H-1} c_k(u), \qquad u^* = \arg\min_{u \in \mathcal{U}} J(u)\]

5.4 PID Residual Correction

A PID loop acts on the instantaneous tracking error to correct any residual model-plant mismatch:

PID error terms
\[e_t = r_t - y_t, \qquad I_t = \text{clip}\!\left(I_{t-1} + e_t,\; -5,\; 5\right), \qquad D_t = e_t - e_{t-1}\]
PID output
\[\text{PID}_t = p\,e_t + i\,I_t + d\,D_t\]

5.5 Output Smoothing & Rate Limiting

The MPC and PID outputs are blended, then smoothed and rate-limited before being sent to the plant:

MPC + PID blend
\[\text{raw}_t = u^* + 0.3 \cdot \text{PID}_t\]
Exponential moving average (EMA) smoothing
\[\text{out}_t = 0.75\,u_{t-1} + 0.25\,\text{raw}_t\]
Rate limiting and saturation
\[u_t = \text{clip}\!\left(\text{out}_t,\; u_{t-1} - 0.25,\; u_{t-1} + 0.25,\; -2,\; 2\right)\]

5.6 Full Execution Flow

Step 1 — State ingestion
Append \(x_t = [v_\text{ego}, a_\text{ego}, \text{roll}, y_t]\) to buffer (keep last 3 steps).
↓
Step 2 — Lifting
Scale → delay-embed (reverse) → polynomial features → \(z_t \in \mathbb{R}^{90}\).
↓
Step 3 — Target assembly
Pull next H targets from future_plan.lataccel; pad last value if needed.
↓
Step 4 — Random-shooting MPC
Evaluate 30 candidate steers × 8 Koopman rollout steps. Select \(u^*\) with minimum J.
↓
Step 5 — PID correction
Compute e, I, D from tracking error. Blend: raw = u* + 0.3·PID.
↓
Step 6 — Smooth + clip
EMA filter (α=0.75) → rate limit (±0.25/step) → saturate to [−2, 2].
↓
Output
Return \(u_t \in [-2, 2]\) to TinyPhysics.

6 · AR-Model Incremental MPC (ar_mpc_model.py)

Design Philosophy: Rather than tracking future lataccel targets with MPC (which causes destructive Δu spikes due to model mismatch), this controller uses an AR(1) model only for disturbance rejection. Target anticipation is handled entirely by a separate feedforward term.

6.1 System Identification: AR(1) Model

The vehicle's lataccel dynamics are identified as a first-order autoregressive model from closed-loop PID rollout data:

AR(1) identified plant model
\[\text{la}[t+1] = \underbrace{0.9577}_{\text{AR\_A}}\,\text{la}[t] + \underbrace{0.0605}_{\text{AR\_B}}\,\text{steer}[t] + \underbrace{0.0404}_{\text{AR\_C}}\,\text{roll}[t] + \underbrace{(-0.007828)}_{\text{AR\_D}}\]

The large AR_A (≈0.958) reflects the slow dynamics of lataccel — it barely changes between steps. AR_B ≈ 0.06 is the identified steer-to-lataccel gain at short horizons.

6.2 Error-Space Reformulation

The controller works in error space. Define:

Error and incremental control
\[e[t] = \text{la}[t] - r[t], \qquad \Delta u[t] = \text{steer}[t] - \text{steer}_\text{ff}[t]\]
Feedforward gain (steady-state calibrated)
\[\text{steer}_\text{ff}[t] = \text{FF\_GAIN} \cdot r_\text{eff}[t], \qquad \text{FF\_GAIN} = 0.30\]
Corrected input gain (so free term → 0 at steady state)
\[b_\text{correct} = \frac{1 - \text{AR\_A}}{\text{FF\_GAIN}} = \frac{1 - 0.9577}{0.30} \approx 0.1410\]
Error-space dynamics
\[e[t+1] \approx \text{AR\_A}\cdot e[t] + b_\text{correct}\cdot \Delta u[t] + \text{free}[t]\]
Free term (disturbances only — roll and bias)
\[\text{free}[k] = \text{AR\_CROLL}\cdot \text{roll}[k] + \text{AR\_D} = 0.0404\cdot\text{roll}[k] - 0.007828\]

6.3 Closed-Form QP Solution

The MPC problem over horizon H=20 is formulated as a QP with state \(e[k]\), control \(\Delta u[k]\), and cost matrices built from the AR(1) model. All gain matrices are precomputed once at class definition — per-step cost is a single dot product.

Prediction matrices
\[S_d[i,j] = a^{i-j}\;\text{ for } j \leq i,\;\text{else } 0 \qquad \text{(lower triangular)}\] \[S_u = b_\text{correct} \cdot S_d, \qquad S_x = [a^1, a^2, \ldots, a^H]^\top\]
Prediction equation
\[E = S_x\,e_0 + S_u\,\Delta U + S_d\,\mathbf{f}\] \[\text{where } E, \Delta U, \mathbf{f} \in \mathbb{R}^H\]
Cost function (tracking + jerk + effort)
\[J(\Delta U) = Q\,\|E\|^2 + R_\Delta\,\|D\,\Delta U - \Delta u_{t-1}\,e_1\|^2 + R_e\,\|\Delta U\|^2\] \[Q = 50, \quad R_\Delta = 7, \quad R_e = 0.02\]
QP optimality matrix
\[M = S_u^\top Q\, S_u + R_\Delta\,(D^\top D) + R_e\, I, \qquad M \in \mathbb{R}^{H \times H}\]
Precomputed MPC gain vectors (first-element extraction)
\[K_{e_0} = -M^{-1} S_u^\top Q\, S_x \in \mathbb{R}^H \;\Rightarrow\; k_{e_0} = K_{e_0}[0]\] \[K_\text{free} = -M^{-1} S_u^\top Q\, S_d \in \mathbb{R}^{H \times H} \;\Rightarrow\; k_\text{free} = K_\text{free}[0,:] \in \mathbb{R}^H\] \[K_{u_\text{prev}} = M^{-1} R_\Delta D^\top e_1 \in \mathbb{R}^H \;\Rightarrow\; k_{u_\text{prev}} = K_{u_\text{prev}}[0]\]
Per-step MPC decision (O(H) dot product only)
\[\Delta u^*[0] = k_{e_0}\,e_0 + k_\text{free} \cdot \mathbf{f} + k_{u_\text{prev}}\,\Delta u_{t-1}\]

6.4 Feedforward + MPC + PID Composition

Additional integral and derivative corrections improve steady-state accuracy and damp rapid error changes:

Integral and derivative corrections
\[\delta_u = \Delta u^*[0] - I_\text{GAIN}\cdot I_t + D_\text{GAIN}\cdot (e_t - e_{t-1})\] \[I_\text{GAIN} = 0.020, \quad D_\text{GAIN} = -0.05\]
Total steer output
\[\text{raw}_t = \text{steer}_\text{ff} + \delta_u\]
EMA + rate limit + saturation
\[\text{out}_t = 0.87\,u_{t-1} + 0.13\,\text{raw}_t\] \[u_t = \text{clip}(\text{out}_t,\; u_{t-1} - 0.07,\; u_{t-1} + 0.07,\; -2,\; 2)\]

6.5 Full Execution Flow

Step 1 — Feedforward
Blend current target with 10-step future mean (70/30). Compute steer_ff = FF_GAIN × r_eff.
↓
Step 2 — Error + Integral + Derivative
e₀ = la − target. Update I (anti-windup ±3). Compute d_error = e₀ − e_prev.
↓
Step 3 — Free vector
Build f ∈ ℝ²⁰ from future roll sequence + constant bias.
↓
Step 4 — MPC (dot product)
Δu* = k_e0·e₀ + k_free·f + k_uprev·Δu_prev (precomputed gains, O(H) cost).
↓
Step 5 — I + D correction
δ_u = Δu* − I_GAIN·I + D_GAIN·d_error.
↓
Step 6 — Output
raw = steer_ff + δ_u → EMA(0.87) → rate limit(±0.07) → clip to [−2, 2].

7 · Feedforward + PID Controller (feedforward_pid.py)

Best performer: Despite its simplicity, this controller achieves the lowest total cost (61.62) — demonstrating that a well-tuned feedforward + PID with preview can be highly competitive with more complex model-based approaches.

7.1 Preview Feedforward

The controller blends the current target with a 10-step future average to anticipate upcoming turns:

Preview-blended effective target
\[r_\text{eff}[t] = (1 - \beta)\,r[t] + \beta\,\frac{1}{N}\sum_{k=1}^{N} r[t+k], \qquad \beta = 0.949,\; N = 10\]
Feedforward steer command
\[u_\text{ff}[t] = \text{FF\_GAIN} \cdot r_\text{eff}[t], \qquad \text{FF\_GAIN} = 0.31498\]

The high future weight (β ≈ 0.95) means the controller is predominantly looking ahead at the average near-future lataccel, not the instantaneous current target. This is important for smooth operation.

7.2 PID Residual Control

After the feedforward handles steady-state, PID corrects residual tracking errors:

PID error accumulation
\[e_t = r[t] - y[t]\] \[I_t = \text{clip}(I_{t-1} + e_t, -5, 5)\] \[D_t = e_t - e_{t-1}\]
PID output
\[u_\text{PID}[t] = \underbrace{0.970\,e_t}_{P} + \underbrace{0.076\,I_t}_{I} + \underbrace{0.5\,D_t}_{D}\]
Combined raw output
\[\text{raw}[t] = u_\text{ff}[t] + u_\text{PID}[t]\]

7.3 Output Smoothing & Rate Limiting

EMA smoothing
\[\text{out}[t] = 0.85\,u[t-1] + 0.15\,\text{raw}[t]\]
Rate limit + saturation
\[u[t] = \text{clip}\!\left(\text{out}[t],\; u[t-1] - 0.2,\; u[t-1] + 0.2,\; -2,\; 2\right)\]

Analysis

AR-MPC's weakness: The AR(1) model gain (AR_B ≈ 0.06) severely underestimates the true steady-state steer-to-lataccel gain (≈0.14 implied by FF_GAIN=0.30). When the MPC tries to use future target changes in the cost, it computes optimal Δu values that are ≈10× too large, immediately hitting the rate limiter every step, making MPC useless. The fix is to restrict MPC to disturbance rejection only — but this still leaves a 2× lataccel cost vs Koopman.

Koopman MPC's advantage over AR-MPC: The 90-dimensional Koopman model captures higher-order dynamics and interactions that the scalar AR(1) completely misses. However, random-shooting MPC with only 30 candidates is a coarse optimiser, and the jerk cost (21.4) suffers from the MPC not perfectly coordinating multi-step actions.

Why FF-PID wins: The high future-weight (β=0.95) feedforward effectively acts as a 10-step preview controller. Since the cost function rewards lataccel accuracy 50× more than jerk, and since TinyPhysics has its own lataccel continuity constraint (±0.5/step), the preview feedforward alone handles most of the tracking. The PID then mopts up model mismatch. Simpler models sometimes win when the dominant disturbance is predictable (known future targets).

10 · Design Trade-offs & Key Insights

Delta-Blended Prediction

The dual-head blending — combining absolute and delta predictions — is a key design choice that improves rollout stability. Pure absolute prediction drifts due to bias in \(C[0]\); pure delta accumulates integration error. The 80/20 blend (α=0.8 favouring delta) keeps predictions anchored to the current output while using the absolute head as a soft reset.

Random Shooting vs. Closed-Form MPC

The Koopman controller uses random-shooting (enumerate 30 constant-hold candidates) rather than solving the QP analytically. With a 90-dimensional state, the AR-MPC's precomputed gain trick only works because it reduces to a scalar error-state. For the full Koopman model, computing \(M = (S_u^\top Q S_u + R)^{-1}\) would require \(S_u \in \mathbb{R}^{H \times n_z}\) — a 90-column matrix — and recomputing every step due to the nonlinear state. Random-shooting is fast enough (30 × 8 = 240 matrix-vector products per step) and avoids the derivation complexity.

Why Not Neural MPC?

Koopman occupies a sweet spot: richer than AR(1) but interpretable (linear A, B, C matrices). Neural MPC would require backpropagation through the ONNX physics model or training a differentiable surrogate — significantly more complexity without guaranteed improvement given the ±0.5 lataccel noise floor.

Spectral Radius vs. Tracking Speed

A tighter spectral radius (ρ=0.97 vs 0.99) means the lifted state modes decay faster in open-loop, reducing error propagation in long rollouts. The trade-off is that very slow system modes are truncated, potentially losing some long-range dynamics. For H=8 steps (0.8 seconds), this is acceptable — \(0.97^8 \approx 0.78\) (22% decay per horizon).

The Preview Advantage

All three controllers exploit the 5-second future plan. The FF-PID with β=0.95 preview essentially steers towards where the car needs to be 1 second from now — a feedforward advantage that is hard to beat when future targets are exactly known.

Key Takeaway: In this challenge, the most impactful improvements come from (1) accurate steady-state feedforward calibration, (2) aggressive future-target preview, and (3) output smoothing/rate-limiting to manage jerk. The Koopman model provides a better plant model than AR(1) but the 50× lataccel weight means tracking accuracy is the primary battleground — and even a simple previewed feedforward handles that well.
